# Architecture Conventions

## Rules

- Use `@/` imports for source files. The alias maps to `./src/*`.
- Prefer `.astro` components for static and server-rendered UI.
- Use React for stateful UI, client interactivity, and isolated Lab experiences
  that benefit from a client-side application surface.
- Content collections are defined in `src/content.config.ts`.
- Query content with `getCollection()` from `astro:content`.
- Filter drafts before rendering public listings.
- Do not edit `dist/` as source. It is generated by `pnpm build`.

## Layout Hierarchy

- `BaseLayout.astro`
- `PageLayout.astro`
- `ProjectLayout.astro`
- `WritingLayout.astro`

## Related Files

- `src/content.config.ts`
