# Personal Website 2026 Brief

## Current Website

- <https://patrickmorgan.org/>
- Can view screenshots in /references

## Website goals

- Impress design and technology leaders. Get them to think "I should hire Patrick"
- Make me a prime candidate to get hired by Anthropic

## Overall guidance

- I want to start by seeding this site with the content I already have, however it will evolve over time
- I want this to be easy to iterate on
- I'm open to changing anything that would better serve the stated goals above.
- I already write and version my blogs and my portfolio project summaries in markdown, so I would like to be able to continue that as much as possible. Also, a lot of the work that I do related to AI gets captured in markdown (skills, rules, docs, etc...), so we should prepare to be able to share those assets when they come around.
- I still want a good amount of flexibility in terms of designing custom views, styles, and interactions so that I can create a standout site
- I'm thinking of this as professional portfolio site first, public knowledge base second

## What's in this directory

- Image assets in `/assets`
- Example blogs in `/blogs`
- Starter project content in `/projects`
- Reference content including screenshots, relevant urls, and more in `/references`
- `planning.md` for future ideas

## Things to highlight

- My expertise: a technical designer, deep expertise in cybersecurity and enterprise, prototyping in code
- My blended background: starting as a frontend engineer and then becoming a designer
- My philosophical and creative nature that I express through my writing

## Content structure

- Home page that acts as an overview and pitch for me as a designer, writer, and technology leader
  - Hero
    - Logos of companies I've worked for
    - My name + photo
    - A headline of what I am/offer
    - Social links: X, LinkedIn, Newsletter
    - Email
  - Projects
    - Professional project case study highlights
    - AI experiment case study highlights
  - Unknown Arts
    - Summary
    - Writing samples
  - Social proof
- Career timeline / story page
- Dedicated project pages for case studies:
  - Professional projects
  - Personal projects
- Blog
  - I want to be able to easily write and manage blog posts via markdown
  - This will be home for my writing even if I move off Substack at some point
  - This may contain articles I send to my newsletter and other blog posts I want to publish but that don't quite fit the newsletter

## Other features

- Embed Figma Slides -> I prepare Figma slide decks for main portfolio pieces and want to embed them directly.
- Embed YouTube videos -> I also record videos of myself presenting those slide decks and want to embed those as well
- Embed Substack sign up -> I want people to subscribe to my writing on Substack as that is my primary touchpoint for publishing

## Tech stack

- I will build this primarily with Claude Code or similar agentic coding tools
- I want to use common tech, highly documented, that I can understand (easy to use, easy to debug, AI-native, lots of ready-made scaffolding)
- I want to focus on the content and unique design details that will stand out when trying to get hired
- Some tech I might consider
  - [ShadCN](https://ui.shadcn.com/), [BaseUI](https://base-ui.com/), [Tailwind CSS](https://tailwindcss.com/)
  - [Hugo](https://gohugo.io/), [Vite](https://vite.dev/), [Astro](https://astro.build/)
  - Deploying to Github pages
- It would be nice to be able to reduce the need for local computer setup (perhaps using a devcontainer) and even be able to edit the site via codespaces on Github to make it as accessible as possible for me.
- I want to minimize maintenance costs

## Style preferences

- Connect to the Unknown Arts brand. Leverage style ideas from the visual assets I make for it with Midjourney. Can view examples in /references.
- Should use responsive design best practices. Must look and feel great on both mobile and desktop.
- Minimal, subtle, intentional, but high level of detail on craft and interaction
- Light mode default with dark mode switch
- "Designer-y": whitespace, Minimal, Intentional monospace font
- Example ShadCN preset
  - <https://ui.shadcn.com/create?base=base&preset=aauzlg>
  - This isn't quite right, but at least in a decent ballpark
